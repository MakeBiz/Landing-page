## 3. Структура сайта (8 страниц + бэкенд)

| Файл | URL | Что это |
|------|-----|---------|
| index.html | / | Главная |
| company.html | /company | О компании |
| partners.html | /partners | Партнёрам |
| contacts.html | /contacts | Контакты |
| openclaw.html | /openclaw | Продукт OpenClaw |
| vector.html | /vector | Продукт Vector |
| intdoc.html | /intdoc | Продукт IntDoc |
| bitrix.html | /bitrix | Продукт Bitrix |
| api/lead.js | /api/lead | Serverless-функция: приём заявок → Telegram |
| vercel.json | — | cleanUrls:true (URL без .html), trailingSlash:false |

**ВАЖНО про навигацию:** index/company/partners/contacts — это копии одного шаблона.
Внутри каждой есть 4 «вкладки» (home/about/partner/contact-page), переключаются JS.
В каждом файле своя вкладка помечена `active` в разметке. JS (`applyRoute`) читает имя файла
и URL, открывает нужную вкладку. Меню использует чистые URL (/company, /partners и т.д.).
`cleanUrls:true` в vercel.json отдаёт company.html как /company — поэтому отдельные файлы
важны (раньше были 404 на /partners и /contacts, потому что физических файлов не было).

## 5. Формы заявок → Telegram

Три формы: client («Обсудить проект»), partner («Стать партнёром»), contactpage («Контакты»).
JS собирает поля, POST на /api/lead. Функция lead.js форматирует и шлёт в Telegram.

**БЕЗОПАСНОСТЬ — токен НЕ в коде.** lead.js берёт токен и chat_id ТОЛЬКО из переменных
окружения Vercel: `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID`.
Их нужно задать в Vercel: Settings → Environment Variables (все 3 окружения) → Save → Redeploy.
Без них формы показывают «Спасибо», но в Telegram ничего не уходит (функция вернёт ошибку).

**Старый токен бота считается скомпрометированным** (засветился в переписке) —
заказчику рекомендовано перевыпустить его через @BotFather (/revoke) и вписать новый
в переменную окружения. НИКОГДА не вписывать токен в код или .md файлы (публичный репозиторий!).

   - Бейдж «Популярный» на IntDoc и Vector больше не обрезается (overflow:visible у featured + padding-top контейнера).
